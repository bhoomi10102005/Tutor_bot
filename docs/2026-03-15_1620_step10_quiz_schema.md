## DB Schema / Migration Changes

Migration added:
- Alembic revision: `8553dfd2f555`
- Down revision: `8d5a35927e58`

Tables created:
- `quizzes`
  - stores quiz metadata, generation spec JSON, total marks, optional time limit, model used, and owner
- `quiz_questions`
  - stores ordered quiz questions with question type, prompt, options JSON, correct answer JSON, marks, and explanation
- `quiz_question_sources`
  - stores chunk/document citations for each quiz question
- `quiz_attempts`
  - stores per-user quiz attempt sessions, timing, score, total marks, and summary JSON
- `quiz_attempt_answers`
  - stores submitted answers per question inside an attempt

Constraints added:
- unique `(quiz_id, question_index)` on `quiz_questions`
- unique `(question_id, chunk_id)` on `quiz_question_sources`
- unique `(attempt_id, question_id)` on `quiz_attempt_answers`
- non-negative checks for quiz marks, question marks, awarded marks, attempt score, attempt total marks, and attempt time spent
- positive-or-null check for `quizzes.time_limit_sec`
- `score <= total_marks` check on `quiz_attempts`

Indexes added:
- FK indexes generated for quiz ownership and relationship columns:
  - `ix_quizzes_user_id`
  - `ix_quiz_questions_quiz_id`
  - `ix_quiz_question_sources_question_id`
  - `ix_quiz_attempts_quiz_id`
  - `ix_quiz_attempts_user_id`
  - `ix_quiz_attempt_answers_attempt_id`
  - `ix_quiz_attempt_answers_question_id`

## Decisions / Tradeoffs

1. Main quiz entities (`quizzes`, `quiz_questions`, `quiz_attempts`) use UUID-style string primary keys to stay consistent with existing user/document/chat tables.
2. Association/detail rows (`quiz_question_sources`, `quiz_attempt_answers`) use `BIGINT` autoincrement IDs, matching existing citation table patterns.
3. Structured fields (`spec_json`, `options_json`, `correct_json`, `summary_json`, `chosen_json`) use JSON columns so later quiz APIs can store validated payloads without text parsing.
4. Added explicit check constraints now so invalid negative marks/time values are blocked at the database layer before quiz APIs are built.
5. Removed Alembic's false-positive `ix_chunks_embedding` drop from the generated migration so retrieval performance is not regressed by this unrelated schema step.
